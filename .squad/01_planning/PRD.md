# PRD: Startup Radar — MVP v1.0

> 작성: PM 에이전트 | 날짜: 2026-02-21 | 상태: 초안 (승인 대기)

---

## 1. 문제 정의 (Problem Statement)

한국 VC 심사역과 스타트업 창업자는 매일 이런 상황에 놓인다:

> 특정 심사역이 요즘 어떤 분야에 관심 갖는지 파악하려면, Threads를 열고, LinkedIn을 열고, 그 사람 블로그를 따로 북마크해서 확인해야 한다. 매일 수십 명을.

**핵심 고통:**
- 관심 있는 인물의 활동이 Threads, LinkedIn, 개인 블로그 등 **여러 플랫폼에 분산**
- VC/스타트업 뉴스가 어느 매체에서 나왔는지 알기 어렵고 **중복·노이즈가 많음**
- VC 공식 블로그·포트폴리오 소식을 챙기려면 **수십 개 사이트를 순회**해야 함

**현재 대안의 한계:**
- Google 알림: 키워드 기반, SNS 포스트 미지원
- RSS 리더: 설정이 복잡하고 SNS 미지원
- SNS 직접 팔로우: 플랫폼별로 흩어져 있어 관리 불가

---

## 2. 솔루션 (Solution)

**"팔로우하면 모든 채널이 한 곳에"**

Startup Radar는 인물과 기관을 구독하면, 그들의 모든 채널 활동이 하나의 피드로 집약되는 정보 허브다.

- 인물을 등록하면 → 그 사람의 Threads, LinkedIn 포스트가 한 피드에
- 기관(VC)을 등록하면 → 공식 블로그, 뉴스레터 업데이트가 한 피드에
- 키워드를 설정하면 → 국내 스타트업 미디어의 관련 기사가 한 피드에

---

## 3. 타겟 사용자

### Primary: VC 심사역

**특성:** 매일 수십 명의 창업자·동료 심사역 동향을 모니터링해야 함

**사용자 스토리:**
- As a VC 심사역, I want to 내가 팔로우하는 창업자/심사역의 SNS 포스트를 한 곳에서 보고 싶다, so that 여러 플랫폼을 순회하지 않고도 최신 동향을 파악할 수 있다.
- As a VC 심사역, I want to 특정 키워드(예: "시리즈A", "딥테크") 관련 뉴스를 자동으로 모아보고 싶다, so that 매일 미디어를 직접 방문하지 않아도 된다.

### Secondary: 스타트업 창업자

**특성:** 투자자의 관심사와 최근 포트폴리오 방향을 파악해 펀드레이징에 활용

**사용자 스토리:**
- As a 창업자, I want to 내가 미팅하려는 심사역의 최근 관심사를 한눈에 파악하고 싶다, so that 미팅 준비를 더 잘 할 수 있다.
- As a 창업자, I want to 특정 VC의 최근 블로그 포스트와 투자 소식을 모아보고 싶다, so that 해당 VC의 투자 방향을 파악할 수 있다.

### Tertiary: 스타트업 생태계 관심자

**특성:** 업계 동향 학습, 커리어 전환 준비

**사용자 스토리:**
- As a 생태계 관심자, I want to 주요 VC/창업자들의 인사이트를 한 곳에서 읽고 싶다, so that 업계 트렌드를 효율적으로 파악할 수 있다.

## 4. 핵심 기능 (Features)

### P0 — MVP 필수 (없으면 서비스 불가)

#### P0-1. 통합 피드 뷰
- 사용자가 사이트에 접속하면 **수집된 최신 콘텐츠**를 시간순으로 볼 수 있다
- 각 아이템에는: 출처(인물명/기관명), 플랫폼, 요약 텍스트, 원본 링크, 발행 시각이 표시된다
- 피드는 "인물 피드 / 기관 피드 / 뉴스 피드" 탭으로 구분된다

#### P0-2. 뉴스 크롤링 & 표시
- 국내 스타트업 미디어(플래텀, 벤처스퀘어, 스타트업투데이)의 최신 기사를 자동 수집한다
- 수집 주기: 1시간마다
- 중복 기사는 제거된다

#### P0-3. VC 블로그 크롤링 & 표시
- 국내 주요 VC(카카오벤처스, 알토스벤처스 등)의 공식 블로그 포스트를 자동 수집한다
- 수집 주기: 6시간마다

### P1 — 있으면 좋음 (MVP에 포함 가능)

#### P1-1. 키워드 필터
- 뉴스 피드에서 키워드로 필터링할 수 있다 (예: "시리즈A", "AI", "핀테크")
- 로그인 없이 URL 파라미터로 동작 (예: `?keyword=AI`)

#### P1-2. 인물/기관 검색
- 이름으로 특정 인물이나 기관을 검색할 수 있다
- 검색 결과에서 해당 엔티티의 최근 포스트를 볼 수 있다

#### P1-3. 인물 SNS 포스트 표시
- 등록된 인물의 **Threads** 포스트를 수집해 표시한다
- **수집 방식:** Meta 공식 Threads API (앱 등록 + API 키 발급 필요)
- **범위:** 공개 프로필 포스트만 지원 (비공개 계정 제외)
- **LinkedIn 제외:** ToS 명시적 금지 및 법적 집행 사례로 인해 MVP 및 v2에서 제외. 공식 API 파트너십 심사 통과 시 v3에서 재검토.
- **결정 근거:** Backend Lead 기술 검토 완료 (2026-02-21, Open_Questions.md Q2)

### P2 — v2 (이번 MVP 제외)

- 구독/팔로우 기능 (로그인 필요)
- 알림 기능 (이메일/슬랙 웹훅)
- 리포트 자동 생성
- 스타트업 행사 캘린더
- 글로벌 VC 추가 (영어 지원)
- YouTube 채널 모니터링

## 5. Out of Scope (명시적 제외)

이번 MVP에서 **만들지 않는 것:**

- 사용자 로그인/회원가입 — 인증 시스템 없음, 모두 퍼블릭
- 콘텐츠 저장/북마크 기능 — 개인화 없음
- 댓글/인터랙션 기능 — 읽기 전용 서비스
- 모바일 앱 — 웹만
- 알림 시스템 — 단순 피드 뷰만
- 유료화/광고 기능

## 7. 사용자 경험 흐름 (User Flow)

### 첫 방문자
```
사이트 접속
→ 통합 피드 표시 (뉴스 + VC 블로그)
→ 탭으로 "뉴스 / VC 블로그 / 인물" 구분
→ 기사 클릭 → 원본 링크로 이동
```

### 키워드 탐색 (P1)
```
검색창에 키워드 입력
→ 해당 키워드 관련 기사 필터링
→ 결과 피드 표시
```

## 8. 성공 지표 (Success Metrics)

### 정량
| 지표 | MVP 목표 |
|------|---------|
| 크롤링 성공률 | > 95% (일별) |
| 피드 업데이트 지연 | < 2시간 |
| 페이지 로딩 속도 | < 2초 (LCP) |
| 수집된 소스 수 | 최소 5개 미디어/블로그 |

### 정성
- 생태계 종사자 3명이 "매일 쓸 것 같다"고 말하면 Product-Market Fit 초기 신호로 판단

## 9. 제약 조건 (Constraints)

- **크롤링 윤리:** 모든 소스의 robots.txt 준수 필수, rate limit 최소 1초
- **개인정보:** SNS 크롤링 시 개인 데이터 최소 수집 원칙
- **예산:** 무료 티어 활용 (Railway/Vercel 무료 플랜 기준 설계)
- **운영:** 초기에는 수동 소스 등록 (관리자 페이지 없음)
