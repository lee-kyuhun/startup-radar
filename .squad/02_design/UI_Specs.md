## 1. 디자인 시스템

### 1-1. 기본 원칙

- **다크모드 우선 (Dark-first):** 심야 리서치 사용자가 많은 도메인 특성. 라이트모드는 v2에서 검토.
- **정보 밀도 우선:** 스캔 중심 레이아웃. 한 화면에 최대한 많은 아이템을 노출.
- **읽기 전용:** 로그인 없음, 인터랙션은 클릭(외부 링크 이동)과 필터 조작만.
- **신뢰성 신호 강조:** 출처(미디어명/인물명) + 시각(발행 시각)을 모든 피드 아이템에 필수 표시.

### 1-3. 타이포그래피

기술 스택: Next.js + Tailwind CSS → Tailwind 기본 타입 스케일 기준으로 정의.
폰트 패밀리: `Pretendard` (한글 최적화) + fallback `system-ui, -apple-system, sans-serif`

| 역할                 | 크기         | 굵기          | 행간   | 용도                        |
|----------------------|--------------|---------------|--------|-----------------------------|
| `type-title-lg`      | 20px (1.25rem) | 700 (Bold)  | 1.3    | 페이지 타이틀               |
| `type-title-md`      | 16px (1rem)  | 600 (SemiBold)| 1.4    | 카드 제목, 섹션 헤더        |
| `type-body`          | 14px (0.875rem)| 400 (Regular)| 1.6   | 요약 텍스트, 본문           |
| `type-meta`          | 12px (0.75rem) | 400 (Regular)| 1.5   | 날짜, 출처명, 태그 텍스트   |
| `type-label`         | 11px (0.6875rem)| 500 (Medium)| 1.4   | 배지, 탭 레이블             |

## 3. P0 화면 스펙

### 3-1. 헤더 / 글로벌 네비게이션 (GlobalHeader)

#### 레이아웃 다이어그램

```
+------------------------------------------------------------------+
|  [SR] Startup Radar          [검색창___________________] [●●분 전] |
+------------------------------------------------------------------+
  ^Logo (좌)                   ^SearchBar (중앙)          ^StatusBadge(우)
```

- 위치: 화면 상단 고정 (sticky). 스크롤 시 유지.
- 배경: `color-dark-800`, 하단 보더: `border-subtle`
- 높이: 56px (desktop), 52px (mobile)

#### 구성 요소

**Logo**
- 텍스트: "Startup Radar"
- 스타일: `type-title-lg`, `color-white`
- 앞에 레이더 아이콘 (SVG, 20x20, `color-accent-blue`)
- 클릭 시 `/` 로 이동

**SearchBar**
- 너비: 최대 480px, 중앙 배치 (desktop); 전체 너비 (mobile에서 헤더 아래 별도 행)
- 배경: `color-dark-700`
- 보더: `border-subtle`, focus 시 `border-active`
- placeholder: "뉴스, VC 블로그 검색..." (D-2 반영: 인물 제거)
- 텍스트 스타일: `type-body`, `color-gray-300`
- 내부 아이콘: 돋보기 아이콘 (좌측), `color-gray-500`
- 높이: 36px
- 상태:
  - default: 배경 `color-dark-700`, 보더 `border-subtle`
  - focus: 보더 `border-active`, 배경 `color-dark-700`
  - filled: 우측에 X 클리어 버튼 표시
- 동작: Enter 또는 돋보기 클릭 → `/search?q={입력값}` 으로 이동

**FeedStatusBadge**
- 내용: "●  N분 전 업데이트" (● 은 색상으로 상태 표시)
- 텍스트 스타일: `type-meta`, `color-gray-500`
- 상태별 색상:
  - 정상 (1시간 이내): `●` = `color-accent-green`
  - 지연 (1~3시간): `●` = `color-accent-amber`
  - 오류 (3시간 초과 또는 수집 실패): `●` = `color-accent-red`
- 위치: 헤더 우측

#### 컴포넌트 3-2-A. TabNav

> D-2 확정: 인물 탭 제거. 뉴스 / VC 블로그 2탭으로만 구성.

```
+-------------------------------+
| [뉴스]    [VC 블로그]          |
+-------------------------------+
     ^active tab: 하단에 2px solid color-accent-blue
```

- 배경: `color-dark-900` (헤더와 구분 없이 일체형)
- 탭 내용: 뉴스 | VC 블로그
- 탭 텍스트: `type-body`, 400 weight
- 탭 간격: 각 탭 좌우 `space-4` 패딩, 탭 간 `space-2` 갭
- sticky: 스크롤 시 헤더 아래 고정 (top: 56px)

**상태 정의:**

| 상태     | 스타일                                               |
|----------|------------------------------------------------------|
| default  | 텍스트: `color-gray-500`, 하단 보더: 없음            |
| hover    | 텍스트: `color-gray-300`, transition 150ms           |
| active   | 텍스트: `color-white`, 하단 보더: 2px `color-accent-blue` |

- 탭 전환 동작: URL hash 변경 없이 클라이언트 상태로 관리 (SPA 방식)
- 기본 활성 탭: "뉴스"

#### 컴포넌트 3-2-B. FilterBar (P1)

```
+-------------------------------------------------------+
| [AI ×]  [시리즈A ×]  [핀테크]  +키워드 추가           |
+-------------------------------------------------------+
```

- 위치: TabNav 아래, FeedList 위
- 배경: 없음 (페이지 배경 그대로)
- 상하 패딩: `space-3` 상하

**키워드 태그 (FilterTag)**
- 크기: auto-width, 높이 28px
- 보더 반경: `radius-sm`
- 텍스트: `type-label`, `color-white`
- 스타일:
  - 비활성: 배경 `color-dark-700`, 보더 `border-subtle`
  - 활성: 배경 `color-accent-blue` (투명도 20%), 보더 `1px solid color-accent-blue`
- 우측 X 버튼: 16px, `color-gray-500` → hover `color-white`

**+키워드 추가 버튼**
- 텍스트: `type-label`, `color-gray-500`
- 좌측 + 아이콘
- hover: `color-gray-300`
- 클릭 시 인라인 텍스트 입력 모드 전환

| 카드 전체      | 배경: `color-dark-800`, 보더: `border-subtle`, 반경: `radius-md` |
| 카드 패딩      | 상하 `space-4`, 좌우 `space-4`                               |
| 카드 간격      | `space-3` (카드 사이 margin-bottom)                           |
| SourceTag      | 배경: 소스 유형별 (`tag-news`, `tag-vc`), 반경: `radius-sm`, 패딩: `space-1` `space-2`, 텍스트: `type-label` |
| 출처명         | `type-meta`, `color-gray-300`, SourceTag 우측 `space-2` 간격 |
| 날짜           | `type-meta`, `color-gray-500`, 우측 정렬                     |
| 외부 링크 아이콘 | 16x16 SVG, `color-gray-500`, 날짜 우측 `space-2` 간격, hover: `color-accent-blue` |
| 제목           | `type-title-md`, `color-gray-100`, 상단 `space-3` 여백, 최대 2줄, 넘치면 `...` |
| 요약           | `type-body`, `color-gray-300`, 상단 `space-2` 여백, 최대 3줄, 넘치면 `...` |

**상태 정의:**

| 상태    | 스타일                                                |
|---------|-------------------------------------------------------|
| default | 배경: `color-dark-800`, 보더: `border-subtle`        |
| hover   | 배경: `color-dark-700`, transition 150ms ease, cursor: pointer |
| visited | 제목 텍스트: `color-gray-500` (브라우저 visited 처리) |

**동작:**
- 카드 전체 클릭 → 외부 링크를 새 탭(`target="_blank"`)에서 오픈
- 링크 아이콘 클릭 → 동일 (새 탭 오픈)
- 키보드 접근성: `tabIndex=0`, `Enter` / `Space` 키 클릭 동일하게 동작

## 4. P1 화면 스펙

### 4-1. 키워드 필터 UI

> 위치: 메인 페이지 FilterBar (3-2-B) 내에서 동작. 별도 페이지 없음.

#### 동작 흐름

```
[+키워드 추가] 클릭
      ↓
인라인 텍스트 입력 활성화 (FilterBar 내부)
      ↓
입력 후 Enter 또는 다른 곳 클릭
      ↓
FilterTag 추가 + URL 파라미터 업데이트 (?keyword=AI,시리즈A)
      ↓
FeedList 필터링 결과 표시
```

#### FilterTag 인라인 입력 상태

```
+------------------------------------------+
| [AI ×]  [시리즈A ×]  [____________]   |  ← 입력 중
+------------------------------------------+
```

- 입력창: 최소 너비 80px, auto-grow
- 배경: `color-dark-700`, 보더: `border-active`, 반경: `radius-sm`
- placeholder: "키워드 입력..."
- ESC 키: 입력 취소, 포커스 해제
- Enter: 태그 추가 확정
- 최대 태그 수: 5개 (초과 시 입력창 비활성화 + 툴팁: "최대 5개까지 추가할 수 있습니다")

#### URL 파라미터 연동

- 키워드 필터 적용 시 URL: `/?tab=news&keyword=AI,시리즈A`
- 페이지 새로고침/공유 시 동일 필터 상태 유지
- 탭 전환 시 키워드는 초기화 (탭별 독립 필터)

- 텍스트: `type-body`, `color-gray-500`, 중앙 정렬
